# 中文 Prompt

你现在要设计的不是“传统网课 UI”，而是一种“科学纪录片 + 数字展览 + 任务控制台”式的高级科普界面。整体审美必须摆脱培训机构感、SaaS 卡片感和儿童科普感，转而呈现叙事性、工业美学、信息精度和电影级视觉张力。

先确定风格模式：

- 如果用户没有指定，默认使用 `auto-hybrid`
- 如果用户指定了 option，就优先服从该 option
- 如果用户给了新的 benchmark 网站，先判断它是在补强现有风格，还是应升级成新的 option

当前可用风格大类：

- `auto-hybrid`：自动选择主方向，再少量引入辅方向
- `cinematic-narrative`：偏 OceanX，沉浸、暗场、电影感、现场叙事
- `editorial-museum`：偏 Brooklyn Museum，留白、图录感、出版物秩序
- `mission-data`：偏 NASA JPL，任务简报、结构精度、数据可信度
- `image-led-premium`：偏 MasterClass，图像主导、海报感、非对称详情页

总目标：

1. 首页像海报，而不是课程目录。
2. 内容页像数字图录、任务档案或高级详情页，而不是普通文章模板。
3. 图像、正文、术语、图注、数据和 CTA 属于同一套视觉宇宙。
4. 既有电影感，也有编辑感和工程感。

核心规则：

- 必须先选一个主方向，不允许平均混搭所有参考风格。
- 首屏只保留一个主视觉、一组标题和一个主 CTA。
- 用全幅媒体、非对称栅格和滚动节奏建立叙事。
- 每个 section 只承担一个任务。
- 不要一开始就堆卡片、徽章、tab 和小组件。
- 内容页开头必须包含分类、标题、日期、阅读时长、作者或任务信息。
- 正文列宽稳定克制，适合长时间阅读。
- 长文要拆成章节标题、图文块、图注、引语、数据块、术语解释、相关推荐。
- 图片、视频、地图、图示、3D 模型都要像正文叙事的一部分。
- 图注要像展签或科研图注，短、准、安静。

排版规则：

- 标题有展示感，正文有阅读稳定性。
- 字号级差明确，不要所有字都差不多大。
- 中文技术内容统一一个中文字体家族；拉丁字母、数字和公式统一一个配套家族。
- 所有变量、上下标、Greek letters 和公式片段必须走统一渲染规则。

严格避免：

- 网课平台感
- SaaS 卡片墙
- 模板化暗色科技风
- 廉价发光描边
- 首页高级、内页普通
- 装饰性强但没有叙事价值的媒体

当出现新的 benchmark 网站，并且它与现有 option 明显不同，不要擅自吸收进默认模式。先询问用户：

“这个参考更适合补强现有风格，还是要单独升格成一个新的风格 option？”

最终输出时，请确保：

- 第一眼建立世界观
- 长内容仍然高级且可读
- 桌面、平板、手机三档都稳定
- 页面整体更像“数字展览 + 科学纪录片 + 精密内容系统”，而不是“在线课程平台”
