# Benchmark Library

这份文件记录 `advanced-science-ui` 当前吸收过的 benchmark 站点。每一条 benchmark 都必须同时覆盖首页和代表性内容页，不允许只看首屏。

记录 benchmark 时，优先回答 5 个问题：

1. 首页怎样建立世界观？
2. 内容页怎样维持同一套视觉宇宙？
3. 信息、图像、数据、术语如何共存？
4. 它真正的高级感来自哪里？
5. 它应该强化已有 option，还是形成新的风格候选？

---

## OceanX

- 归属判断：强化 `cinematic-narrative`
- 首页学习重点：
  - 全幅媒体先行，标题像纪录片章节名
  - 暗场高对比下的大字排版很有压迫力
  - 滚动不是模块堆叠，而是叙事推进
- 内容页学习重点：
  - 内容页像任务档案，而不是普通文章
  - 图片、地图、段落和任务信息有连续节奏
  - 科学任务信息并不营销化，而是像现场简报
- 对 skill 的核心贡献：
  - 建立“科普首页要像海报”的默认上限
  - 建立“内容页也要保持沉浸气质”的要求
- 风险提醒：
  - 容易被误学成全黑背景 + 发光未来风

## Brooklyn Museum

- 归属判断：强化 `editorial-museum`
- 首页学习重点：
  - 留白、比例、版面秩序本身就是高级感来源
  - 不是靠炫技，而是靠克制的出版物设计
- 内容页学习重点：
  - 展览页像数字图录，文字、图像、元信息关系很安静
  - 说明文字、图注和标题层级处理成熟
  - 适合学习长文阅读的稳定节奏
- 对 skill 的核心贡献：
  - 建立“长内容也必须高级”的编辑基线
  - 建立“图注像展签”的做法
- 风险提醒：
  - 容易被误学成“只是大留白”，却忽略信息密度控制

## NASA JPL

- 归属判断：强化 `mission-data`
- 首页学习重点：
  - 信息分发很清楚，导航克制，模块秩序强
  - 科技感来自结构化，而不是装饰性未来风
- 内容页学习重点：
  - 元信息、日期、任务名、正文、相关推荐属于同一套编辑系统
  - 任务页和新闻页都兼具科研可信度与大众可读性
  - 图像、图示、数据块服务内容，不喧宾夺主
- 对 skill 的核心贡献：
  - 建立“任务简报式容器”和“可信元信息系统”
  - 建立“工业感来自精度”的判断标准
- 风险提醒：
  - 容易被误学成普通 dashboard 或模块化资讯站

## MasterClass

- 归属判断：强化 `image-led-premium`
- 首页学习重点：
  - 以图像质量、裁切和排版位移建立高级感
  - 非对称网格和流媒体内容编排值得吸收
- 内容页学习重点：
  - 详情页的核心不是“讲全”，而是高价值信息的克制组织
  - 人物/主题主视觉对页面气质影响极大
  - CTA 清楚但不过度打断主体验
- 对 skill 的核心贡献：
  - 建立“图像主导型教学页面”的高端做法
  - 建立“海报感内容详情页”的参考上限
- 风险提醒：
  - 容易被误学成纯营销页或普通课程售卖页
