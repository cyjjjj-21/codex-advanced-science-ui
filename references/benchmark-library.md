# Benchmark Library

这份文件记录 `advanced-science-ui` 当前吸收过的 benchmark 站点。每一条 benchmark 都必须同时覆盖首页和代表性内容页，不允许只看首屏。

如果某个站点的高级感明显依赖滚动、hover、sticky 或 reveal，也必须把动态交互语言一起记录下来，而不是只保留静态截图印象。

记录 benchmark 时，优先回答 5 个问题：

1. 首页怎样建立世界观？
2. 内容页怎样维持同一套视觉宇宙？
3. 信息、图像、数据、术语如何共存？
4. 它真正的高级感来自哪里？
5. 它应该强化已有 option，还是形成新的风格候选？

---

## OceanX

- 归属判断：强化 `cinematic-narrative`
- 首页学习重点：
  - 全幅媒体先行，标题像纪录片章节名
  - 暗场高对比下的大字排版很有压迫力
  - 滚动不是模块堆叠，而是叙事推进
- 内容页学习重点：
  - 内容页像任务档案，而不是普通文章
  - 图片、地图、段落和任务信息有连续节奏
  - 科学任务信息并不营销化，而是像现场简报
- 对 skill 的核心贡献：
  - 建立“科普首页要像海报”的默认上限
  - 建立“内容页也要保持沉浸气质”的要求
- 风险提醒：
  - 容易被误学成全黑背景 + 发光未来风

## Brooklyn Museum

- 归属判断：强化 `editorial-museum`
- 首页学习重点：
  - 留白、比例、版面秩序本身就是高级感来源
  - 不是靠炫技，而是靠克制的出版物设计
- 内容页学习重点：
  - 展览页像数字图录，文字、图像、元信息关系很安静
  - 说明文字、图注和标题层级处理成熟
  - 适合学习长文阅读的稳定节奏
- 对 skill 的核心贡献：
  - 建立“长内容也必须高级”的编辑基线
  - 建立“图注像展签”的做法
- 风险提醒：
  - 容易被误学成“只是大留白”，却忽略信息密度控制

## Apple Environment

- 归属判断：强化 `editorial-museum`，并校准 `mission-data`；不新增 option
- 首页学习重点：
  - 世界观不是靠强情绪大片，而是靠“年度目标 + 巨大数字 + 简洁解释”建立公共承诺感
  - 明亮底色、超克制排版和可核验主张同样可以形成海报级压场能力
  - 生命周期内容用重复而稳定的 `Approach / Progress` 结构推进，读起来像一份被精心设计过的公开简报
- 内容页学习重点：
  - FAQ、方法说明、披露页也可以保持高级感，只要阅读列稳定、编号清楚、技术术语和链接属于同一系统
  - PDF 报告、白皮书、脚注和方法论说明不是附件，而是信任感的一部分
  - 内容页的可信度来自组织方式、措辞克制和证据链完整，而不是复杂装饰
- 对 skill 的核心贡献：
  - 证明“高级科普首页”可以是白底、安静、强证据的，不必默认依赖暗场或大片主视觉
  - 给 `editorial-museum` 补上“public-briefing / evidence-first”这一条成熟分支
  - 让 `mission-data` 更像对公众开放的任务报告，而不是内部 dashboard
- 风险提醒：
  - 容易被误学成普通苹果官网营销页，忽略它真正厉害的是承诺、证据、脚注和报告入口的统一编排
  - 也容易被误学成干瘪 FAQ 或合规文档，失去首页的尺度感和内容页的编辑秩序

## MoMA

- 归属判断：强化 `editorial-museum`；不新增 option
- 首页学习重点：
  - 首页不是强压场海报，而是高度机构化的入口系统：展览、馆藏、杂志、访问信息被组织成清楚的文化机构导航
  - 高级感来自成熟而不张扬的排版、克制的留白，以及“你正在进入一个真实机构知识系统”的信任感
  - 首页既服务浏览，也服务访问和检索，不靠大情绪视觉，而靠长期稳定的品牌秩序
- 内容页学习重点：
  - 馆藏作品页会把标题、作者、年份、材质、馆藏编号、楼层、展厅、安装历史等元信息组织成正式标签系统
  - 展览页和杂志页都延续同一种机构化阅读气质：栏目、日期、作者、关联内容和访问信息很安静，但层级极清楚
  - 内容页能兼容“图录式 object page”“展览信息页”“杂志文章页”三种不同内容类型，而不失去同一套视觉宇宙
- 对 skill 的核心贡献：
  - 给 `editorial-museum` 补上“collection-grade metadata / institutional catalog reading”这条成熟分支
  - 证明丰富的元信息不一定会把页面做乱，只要它被当作展签和编目系统来设计
  - 让数字图录页、标本页、档案页更接近真实博物馆网站，而不是普通美术资讯页
- 风险提醒：
  - 容易被误学成普通白底 CMS 或文化机构官网，忽略它真正厉害的是标签系统、作品编目和内容类型的一致性
  - 也容易只抄“安静”和“白底”，却丢掉机构化阅读的精确层级

## Fondation Cartier

- 归属判断：强化 `editorial-museum`；不新增 option
- 首页学习重点：
  - 首页允许比传统 museum 站更品牌化、更海报化：高饱和纯色场、超大标题、强机构 logo 和公开入口可以同时成立
  - `Visit / Programme / Tickets / Shop` 这些面向公众的功能入口不会破坏高级感，只要它们被放进统一的机构排版秩序
  - 首屏像文化机构海报而不是“展览 CMS 列表”，但仍保留时间、地点、活动身份这些正式元信息
- 内容页学习重点：
  - Collection、artist page、architecture page 仍然是典型的机构化内容系统：艺术家、作品、展览、建筑、出版物等内容彼此可编目、可跳转
  - 它不像 The Pudding 那样每篇重做交互机制，而是用稳定的机构框架容纳更鲜明的品牌气质
  - 内容页高级感来自图录逻辑和机构关系网，而不是单纯靠时尚视觉压场
- 对 skill 的核心贡献：
  - 给 `editorial-museum` 补上“高时尚基金会 / 文化机构海报感”这条支线
  - 证明 museum 气质不一定只能安静白底，也可以是强品牌色场 + 正式元信息并存
  - 让文化机构首页能更公开地整合 visit、programme、ticketing 和 shop，而不掉成商业售票页
- 风险提醒：
  - 容易被误学成只剩大色块和时尚排版，丢掉内容页的机构化阅读秩序
  - 也容易被误学成品牌 campaign page，而忽略它本质上仍是 museum-grade content system

## The Pudding

- 归属判断：升级为新 option `playful-data-essay`
- 首页学习重点：
  - 首页不是海报，也不是 museum 式序厅，而是一本可浏览、可筛选、带人格化标签系统的故事档案库
  - 品牌气质来自 sticker、手作感图形、目录式列表和轻松但高度辨识的编辑语气
  - 首页强调“这里是一间持续产出 visual essays 的实验室”，而不是单一世界观的压场首屏
- 内容页学习重点：
  - 每篇内容页都允许围绕一个核心交互机制重建叙事骨架，例如视角切换、地图探索、时间轴滚动、点击取样
  - 数据、插画、动画和说明文字是同等重要的叙事单位，不是正文的附属物
  - `Data and methods`、作者署名、外部资料、打赏入口等元信息会被保留，但不会像传统报告页那样主导气质
  - `happy map` 这类内容页会把 canvas 地图、鼠标指针状态和原位人物卡片做成探索机制的一部分，说明 `hover_behavior` 和 `cursor_interaction` 在这一类里可以承担真正的信息发现功能
- 对 skill 的核心贡献：
  - 补上“高级科普 UI”里偏 visual essay / explorable explanation 的一支，而不是只有 museum、mission 或 cinematic
  - 证明互动机制本身可以成为主叙事，不必总是退回图文章节结构
  - 提供一种更轻、更俏皮、但仍然成熟的编辑型数据故事方向
- 风险提醒：
  - 容易被误学成信息密度失控的花哨网页，或者把 playful 错做成儿童化设计
  - 也容易只抄 sticker 和插画表层，却丢掉真正重要的“一个洞察配一个定制机制”

## Distill

- 归属判断：`scholarly explorable explanation` 新候选；暂不直接并入 `playful-data-essay`
- 首页学习重点：
  - 首页不是海报，也不是 museum 式首页，而是学术解释平台的期刊入口
  - 品牌感来自论文级严肃性、长文结构、作者署名、引用与开放源代码入口，而不是人格化编辑语气
  - 首页世界观强调“这里是解释机器学习思想的研究型出版物”，不是故事档案馆或文化机构
- 内容页学习重点：
  - 代表性内容页把参数滑杆、原位试验台、固定控制条和章节正文揉成同一个解释系统
  - 点击小图不是打开独立详情页，而是把参数、step 和可视化状态同步进上方试验台，属于 `sticky sync / parameter swap`
  - 滑杆与按钮改变的是论证过程本身，不是给正文增加一点点互动装饰
- 动态交互学习重点：
  - `scroll_narrative` 中等：滚动主要推进论证章节，而不是电影镜头感转场
  - `state_transition_style` 高：参数变化、step 推进和局部刷新是主语言
  - `click_state_pattern` 以原位状态机为主，不应误记成“很多小弹窗”
  - `hover_behavior` 不是主骨架，只承担轻量注释和补充说明
- 对 skill 的核心贡献：
  - 证明有些网站的风格差异不在静态视觉，而在“交互本身就是知识表达方式”
  - 给 motion 验证链路补上“原位解释状态机”和“点击后参数同步”的真实样本
  - 说明 `playful-data-essay` 不能覆盖所有 explorable explanation，学术解释型交互可能需要独立分支
- 风险提醒：
  - 容易被误学成普通论文页加几个 slider，忽略它真正厉害的是正文、参数、图示和控制条的统一解释逻辑
  - 也容易被误塞进 `playful-data-essay`，从而把学术解释误做成轻佻故事页

## Apple Scroll Narrative (AirPods Pro 3)

- 归属判断：强化 `cinematic-narrative`，并校准它与 `image-led-premium` 的动态边界；不新增 option
- 首页学习重点：
  - Apple 首页仍以品牌化 hero、固定全局导航和模块化专题入口建立世界观，强调“高完成度品牌宇宙”而不是单一沉浸场景
  - 高级感来自媒体质量、版式收束和品牌秩序，而不是重情绪暗场
- 内容页学习重点：
  - `AirPods Pro 3` 页面像长篇产品故事页：章节推进、媒体分镜、sticky local nav 和多段 highlights gallery 连成一条滚动叙事线
  - 亮色背景、超克制的文案和产品摄影并没有把它推向新流派，它仍是滚动叙事在品牌化语境中的成熟实现
  - 内容页的核心不是复杂数据互动，而是让媒体、卖点和辅助说明随着滚动分批出场
- 动态交互学习重点：
  - `scroll_narrative` 高：滚动承担章节推进和媒体换景
  - `sticky_reveal_behavior` 高：local nav、gallery hash 切换和 section 内 tab 共同组成 staged reveal
  - `click_state_pattern` 以 hash 轮播、tab 切换和局部内容替换为主，不是新的交互哲学
  - `image_strategy` 仍然极强，但图像服务的是 scroll-driven 叙事，不是单纯封面感
- 对 skill 的核心贡献：
  - 证明明亮、品牌化、产品导向的滚动故事页仍可落在 `cinematic-narrative`，不必默认等同 `image-led-premium`
  - 给 `cinematic-narrative` 补上 `sticky gallery / staged media progression / hash-based highlight switching` 这组动态样本
  - 帮助区分“图像主导详情页”和“滚动分镜叙事页”的边界
- 风险提醒：
  - 容易被误学成单纯的大图营销页，忽略它真正厉害的是滚动分镜和局部状态切换的秩序
  - 也容易被误判成新风格，只因为它很会滚动，但它并没有改变阅读哲学

## NASA JPL

- 归属判断：强化 `mission-data`
- 首页学习重点：
  - 信息分发很清楚，导航克制，模块秩序强
  - 科技感来自结构化，而不是装饰性未来风
- 内容页学习重点：
  - 元信息、日期、任务名、正文、相关推荐属于同一套编辑系统
  - 任务页和新闻页都兼具科研可信度与大众可读性
  - 图像、图示、数据块服务内容，不喧宾夺主
- 对 skill 的核心贡献：
  - 建立“任务简报式容器”和“可信元信息系统”
  - 建立“工业感来自精度”的判断标准
- 风险提醒：
  - 容易被误学成普通 dashboard 或模块化资讯站

## MasterClass

- 归属判断：强化 `image-led-premium`
- 首页学习重点：
  - 以图像质量、裁切和排版位移建立高级感
  - 非对称网格和流媒体内容编排值得吸收
- 内容页学习重点：
  - 详情页的核心不是“讲全”，而是高价值信息的克制组织
  - 人物/主题主视觉对页面气质影响极大
  - CTA 清楚但不过度打断主体验
- 对 skill 的核心贡献：
  - 建立“图像主导型教学页面”的高端做法
  - 建立“海报感内容详情页”的参考上限
- 风险提醒：
  - 容易被误学成纯营销页或普通课程售卖页
