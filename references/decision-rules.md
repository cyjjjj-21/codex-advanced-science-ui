# Decision Rules

这份文件定义 `advanced-science-ui` 在面对新 benchmark 时如何判断它的归属和后续动作。

## 判断类型

### 1. 强化已有风格

满足以下情况时，归为“强化已有风格”：

- 与某个现有 option 的核心气质高度一致
- 只是把某条既有能力做得更成熟
- 不会明显改变默认 prompt 的整体方向

动作：

- 更新 `benchmark-library.md`
- 如有必要，微调 `PROMPT.md`
- 不需要询问用户是否新建 option

### 2. 修正已有风格

满足以下情况时，归为“修正已有风格”：

- 它暴露了我们对某一现有风格的理解偏差
- 会影响某个 option 的描述边界或禁忌项
- 仍可归入现有分类，但需要更精确的定义

动作：

- 更新 benchmark library
- 更新相应 option 描述
- 必要时调整 prompt 中相关规则

### 3. 新增风格候选

满足以下任意情况时，归为“新增风格候选”：

- 与现有任一 option 在 3 个以上核心维度上差异明显
- 更像一种新的设计哲学，而不是现有 option 的变体
- 混入默认模式会明显降低输出稳定性
- 有很强的专门适用场景

动作：

- 先询问用户是否要升级为新的 style option
- 未得到明确同意前，只写入 benchmark library，不修改全局分类

## 标准询问语句

当触发“新增风格候选”时，使用统一问法：

“这个参考更适合补强现有风格，还是要单独升格成一个新的风格 option？”

## 默认保守策略

如果判断模糊，优先按“补强已有风格”处理，并在 benchmark 条目里记录不确定性，不要贸然新建 option。
