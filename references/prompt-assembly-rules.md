# Prompt Assembly Rules

`PROMPT.md` 是稳定版 prompt，不是原始经验堆场。更新它时，遵循以下规则：

## 应该进入主 prompt 的内容

- 多个 benchmark 反复印证的稳定规律
- 会明显影响输出质量的高层设计原则
- 对所有或大多数 option 都成立的避免项
- option 调用方式和默认行为

## 不应该直接进入主 prompt 的内容

- 只在一个站点上成立的细节偏好
- 太具体的视觉装饰
- 带有强烈时代性或一次性的页面策略
- 还没有经过至少一次真实任务验证的猜想

## 组织原则

主 prompt 尽量保持 4 层：

1. 总目标
2. 默认模式与风格选择方式
3. 首页 / 内容页 / 信息系统的稳定规则
4. 触发询问用户的条件

## 更新原则

- 如果一条经验只是补充背景，不一定要写进 prompt
- 如果一条经验会改变默认输出方向，才考虑写进 prompt
- 如果一条经验只属于某个 option，就放在 option 或 benchmark 层，不一定写进全局 prompt
