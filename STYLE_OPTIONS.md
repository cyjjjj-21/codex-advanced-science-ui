# 风格选项

`advanced-science-ui` 在 v1.5 之后不再只有一段固定 prompt，而是允许在统一“高级科普 UI”总方向下切换不同风格重心。

默认模式是 `auto-hybrid`。如果用户没有明确指定，skill 会先根据任务目标、内容类型和参考站来自动判断最合适的主方向；如果遇到明显异类参考，则先询问用户要不要升级成新的 option。

## 当前可用 option

### `auto-hybrid`

- 适用：用户没有明确指定风格，希望自动综合电影感、编辑感和工程秩序感
- 特征：不平均混搭；优先根据内容任务自动选择一个主方向，再带少量辅方向
- 默认边界：除非题目明显要求“互动式 visual essay / 可玩数据故事 / 杂志化数据叙事”，否则不要主动混入 `playful-data-essay`
- 默认推荐：大多数科普、教学、研究传播页面

### `cinematic-narrative`

- 主要来源：OceanX
- 补强来源：Apple Scroll Narrative（代表页：AirPods Pro 3）
- 适用：沉浸式、电影化、滚动叙事强、主视觉压场的页面；也适合明亮、品牌化、但依然靠 scroll / sticky / staged media progression 推进叙事的专题页
- 关键词：深色、全幅媒体、高对比、视差、任务现场感
- 动态边界：允许 scroll-driven 场景推进、层次 reveal、sticky local nav、hash-based gallery 切换和章节过场，但必须服务叙事，不能把读者变成被特效拖着走
- 避免：做成廉价未来风、过量发光、满屏视觉噪音

### `editorial-museum`

- 主要来源：Brooklyn Museum
- 补强来源：Apple Environment、MoMA、Fondation Cartier
- 适用：安静、留白重、图录感、编辑设计感强，或需要“公共事务 brief + 数字图录 + 馆藏目录级元信息”气质的长内容页面；也适合更品牌化、更海报化的文化机构首页
- 关键词：出版物感、静态高级、克制、严谨、策展感、证据导向、馆藏标签系统、机构化阅读、品牌化色场、文化机构海报感
- 动态边界：动效最克制，只允许轻 reveal、轻 hover 和低噪音状态切换；不允许强鼠标跟随、夸张 parallax 或噪音式滚动特效
- 避免：为了“高级”把页面做得过于空、过于冷、信息承载力不足

### `mission-data`

- 主要来源：NASA JPL
- 适用：科学任务、研究报告、数据面板、工程可信度要求高的页面
- 关键词：模块秩序、元信息清晰、任务简报、数据可信、技术冷静、方法论透明
- 动态边界：动效应服务信息揭示、步骤推进和参数展开，不做炫技镜头，不把页面演成科幻片
- 避免：做成 SaaS 仪表盘、营销数据卡、徽章堆叠，或只剩“理工感”却缺少可核验依据

### `image-led-premium`

- 主要来源：MasterClass
- 适用：高质量人物或主题摄影主导、需要海报感和高级转化感的教学/课程页面
- 关键词：图像主导、非对称、流媒体感、主题感强、排版位移
- 动态边界：偏图像主导 hover、轻量 details reveal 和封面级转场，不把交互机制做成主骨架
- 避免：变成普通课程卡片墙，或只有封面高级、内容页失速

### `playful-data-essay`

- 主要来源：The Pudding
- 适用：需要互动式 visual essay、杂志化数据故事、读者参与、定制可视化隐喻的页面
- 关键词：定制交互、explorable explanation、轻俏但不幼稚、数据叙事、可玩结构、手作编辑感
- 动态边界：允许交互机制本身成为叙事骨架，例如切换视角、逐步探索、拖拽或点击取样，但每个机制都必须带来明确的信息增量
- 避免：把页面做成儿童科普、花哨玩具、拼贴噪音，或只剩 gimmick 没有清晰洞察
- 额外边界：如果页面语气明显偏学术说明、交互更像参数试验台或原位解释状态机，而不是杂志化故事体验，不要勉强并入这一类，应先视为新候选

## 调用建议

可以这样触发：

```text
用 $advanced-science-ui 做这个页面
```

也可以显式指定：

```text
用 $advanced-science-ui，偏 mission-data
用 $advanced-science-ui，偏 editorial-museum，但保留一点电影感
用 $advanced-science-ui，偏 playful-data-essay，做一个可交互的数据故事页
```

## 何时需要新增 option

如果新参考网站满足以下任意情况，就不应该直接硬塞进现有分类，而应先询问用户：

- 在 3 个以上核心维度上与现有 option 明显不同
- 更像一种新的设计哲学，而不是现有风格的变体
- 会明显改变默认 `auto-hybrid` 的整体输出气质
- 场景非常专门，直接混入默认模式会降低稳定性
- 它的动态交互语言已经从“点缀页面”升级成新的阅读范式

对应的标准问法是：

“这个参考更适合补强现有风格，还是要单独升格成一个新的风格 option？”
