# 高级科普 UI

## 概览

把科学传播和教学界面当成一套完整的编辑系统来设计：首页要有电影海报感，内容页要能承载长文、图像、数据和术语，而且始终保持同一套审美气质。

如果当前任务已经会用 `frontend-skill`，把它当作基础实现层；本 skill 负责把视觉方向拉向“数字展览、任务档案、纪录片式叙事”，避免做成普通课程平台。

## 气质选择

先选一个主气质，不要平均混搭：

- `OceanX`：沉浸、深色、全幅媒体、高对比、电影感科学叙事
- `Brooklyn Museum`：编辑感、留白、安静、高级图录气质
- `NASA JPL`：精密、模块化、任务控制台、科研可信度
- `MasterClass`：图像主导、海报感、非对称、高级转化页

## 首页规则

- 把首屏当海报，不要当课程目录。
- 只保留一个主视觉平面：人物、装置、环境、实验场景、仪器或视频。
- 首页第一屏控制在一个标题、一句副标题、一个主 CTA 的密度。
- 每个 section 只做一件事：解释、证明、揭示、深化或转化。
- 先用裁切、留白、比例、对齐、字级和滚动节奏建立高级感，再考虑装饰性 UI。
- 优先使用全幅段落、非对称栅格、叙事滚动，避免一上来就是卡片墙。

## 内容页规则

- 内容页必须延续首页的世界观，不能掉回普通 CMS 文章模板。
- 开头必须给出内容身份信息：分类、标题、日期、阅读时长、地点、作者、任务名、时长等。
- 正文列宽要稳定克制，不要满屏摊开。
- 长内容要拆成清晰的编辑单元：
  - 章节标题
  - 图像或媒体块
  - 图注
  - 引语
  - 关键数据
  - 术语解释
  - 相关推荐
- 图片、视频、地图、示意图、3D 内容都要像正文叙事的一部分，而不是附件。
- 图注要像展签或科学图注：短、准、安静。
- 元信息和数据块要像任务简报，不要做成营销 badge。

## 排版与信息系统

- 标题要有展示感，正文要有阅读稳定性。
- 标题、导语、正文、标签、图注、指标数值之间要有明显的字号级差。
- 全站技术标签必须统一，不允许每个模块各写一套。
- 如果出现变量、上标、下标、希腊字母或公式片段，必须走同一套数学感渲染系统。
- 中文技术 UI 中，中文建议统一一个字体家族，拉丁字母和数学文本统一一个配套家族，除非项目已有更强设计系统。

## 必须避免

- 网课平台感
- SaaS 仪表盘式卡片拼盘
- 廉价未来风发光边框
- 过多的 pills、徽章、tab 和小控件
- 首页高级、内页普通的“两张皮”
- 只有装饰作用、没有叙事作用的媒体

## 自检问题

- 首页是不是第一眼就建立了世界观？
- 内容页能不能撑住长阅读而不散？
- 元信息、图像、数据和正文是不是像同一套系统？
- 去掉阴影和渐变以后，还高级吗？
- 在桌面、中间宽度和手机上，是否依然稳定、安静、可读？
